# Advanced Tuition Centre Management System 📚💻

## 📌 Project Overview
This project is an **Object-Oriented Programming (OOP)** application developed during **Semester 2** at **Asia Pacific University (APU)**. The system is designed to digitalize the operations of a tuition centre, enhancing efficiency in managing student registrations, class scheduling, and staff records.

The system utilizes **Java Swing** for the Graphical User Interface (GUI) and follows strict OOP principles to ensure code reusability and maintainability.

## 🛠 Tech Stack & Tools
* **Language:** Java (JDK 17+)
* **GUI Framework:** Java Swing / NetBeans GUI Builder
* **Data Storage:** Text-based Database (.txt files for data persistence)
* **IDE:** NetBeans / IntelliJ IDEA

## 🏗 Key OOP Concepts Applied
* **Encapsulation:** Used to protect user data and maintain class integrity.
* **Inheritance:** Implemented to share common attributes among different user roles (Admin, Receptionist, Tutor, Student).
* **Polymorphism:** Utilized for flexible method overriding across different system modules.
* **Abstraction:** Defined core system functionalities while hiding complex implementation details.

## 👥 User Roles & Functionalities
The system is divided into four distinct modules, each with specific logic and permissions:

### 1. Administrator Module
* **System Management:** Registering and managing records for Tutors and Receptionists.
* **Reporting:** Generates analytical reports on tuition centre performance.

### 2. Receptionist Module
* **Student Lifecycle:** Handles student registration, enrollment, and profile updates.
* **Payment Management:** Processes tuition fees and issues receipts.

### 3. Tutor Module
* **Class Management:** View assigned teaching schedules and student lists.
* **Academic Updates:** Updates student attendance and grading records.

### 4. Student Module
* **Personal Dashboard:** View enrolled subjects, class schedules, and academic results.

## 📂 Project Structure
* `/src`: Contains the Java source code categorized by user roles.
* `/data`: Text files used for data persistence (storing system information).
